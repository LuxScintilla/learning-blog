Object Literal

                      
//Object Literal
const me = {};
//Object Literal is a shorthand for this
const me = new Object();

Add Properties to an empty Object

                      
//Empty Object
const person = {};
//Add Properties to the Object
person.name = "Lux";
person.age = 35;

Regular Object with a Method

                      
//Regular Object
const person = {
  talk() {
    return "Talking";
  },
};
//Create Object from Regular Object. person is now the prototype of me.
const me = Object.create(person);

Regular Object with a Method, another way to create a new object by using Object.assign 

                      
//Regular Object
const person = {
  talk() {
    return "Talking";
  },
};
//Use the Object.assign() to create a new Object, with the propeties and methods of the original Object Literal copied to the new one. This however does not do prototype.
const me = Object.assign({}, person);

Regular Object with Method, used to set Inheritance to another Object. 

                      
//Regular Object
const person = {
  talk() {
    return "Talking";
  },
};
//Create Object and set Inheritance from another Object
const me = {};
Object.setPrototypeOf(me, person);

Regular Object with Method being used in a Factory Function 

                      
//Regular Object
const myCoolProto = {
  talk() {
    return `Hello, I am ${this.name}`;
  }
}
//Factory Function creates a new object, with the new object inheriting the myCoolProto properties instead of having them on the new object itself, by using the Object.create within the Factory Function
function createPerson(name) {
  return Object.create(myCoolProto, {
    name: {
      value: name
    }
  })
}

Constructor Function

                      
//Constructor Function
function Person(name) {
  this.name = name;
}
//Create Object from Constructor Function
const me = new Person("Lux");
//Add a Method to the Person Constructor Function, which me can then inherit from Person
Person.prototype.talk = function () {
  return "Talking";
};
//Object.prototype ... is a property on the constructor function, which means the children of that Constructor Function will inherit those properties
//__proto__ on the child ... points to the parent object that it is inheriting from

Constructor Function

                      
//Constructor Function 
function Person(name) {
  this.name = name;
  this.age = age;
}
//Constructor Function written using the Object.assign()
const Person = function(name) {
  Object.assign(this, {name, age});
}

Factory Function

                      
//Factory Function
function createPerson(name) {
  return {
    name: name,
    talk() {
      return `I am ${name}`;
    },
  };
}
//Create Object with Factory Function
let me = createPerson("Lux");

// OR By returning the object Variable
function createAgain(name) {
  const object = {
    name: name,
    talk() {
      return `I am ${name}`;
    },
  };
  return object;
}
//Create Object with Factory Function
let meAgain = createAgain("Lux");

Classes

                      
//Classes are really Constructor Functions under the hood
//a Factory is a function that returns an object, while a Class is a template for an object
//Using a Class
class Person {
  talk() {
    return "Talking";
  }
}
//Create Object from Class
const me = new Person();

Using a Class which also has a Constructor

                      
//Using a Class which also has a Constructor
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  talk() {
    return "Talking";
  }
}
//Create Object from Class
const me = new Person("Lux", 35);
//Class Extend with another Class
class SuperHuman extends Person {
  fly() {
    return "Flying";
  }
}
const you = new SuperHuman();

Modules and Bundles

The 2 modules I done this week were daunting at the start when I just seen them for the first time. Now at the end of the week, they make sense I have gone through the tutorials provided by TOP and watched several videos on the topic on Youtube. I will from now on start using npm and webpack in this personal project too. This way I will ensure I know what I am doing otherwise I mess up everything I have created here up till now.

When you install Node.js, npm comes along with it.
Go to npmjs.com to browse for modules/packages.

To create the "package.json" file

npm init -y

To install a new module/package as a project dependency

npm install name-of-package --save

To install a new module/package as a DEV dependency, such as webpack webpack-cli

npm install name-of-package --save-dev

To install only the dependencies used inside a project

npm install --production

Uninstalling dependencies is easy (uninstall/remove/rm/un)

npm uninstall name-of-module

Installing specific versions of modules

npm install name-of-module@x.x.x

Update module to the latest version

npm update name-of-module

Installing a Global Module. This does not install the module into the node_module folder in your project directory, you can check where it installs instead with the below command

npm install -g name-of-module
npm root -g

Scripts in package.json
Then use command "npm run build"

"scripts": {
"build": "webpack"
}

Create a file called "webpack.config.js" 

                    
const path = require("path")

module.exports = {
  mode: "development",
  entry: {
    bundle: path.resolve(__dirname, "src/index.js"),
  },
  output: {
    path: path.resolve(__dirname, "dist"),
    filename: "[name].js",
  },
  module: {
    rules: [{
      test: /\.css$/,
      use: ["style-loader", "css-loader"],
      },
    ],
  },
}
